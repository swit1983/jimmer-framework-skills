# Jimmer 框架核心特性

Jimmer 是 JVM 中最先进的 ORM，同时面向 Java 和 Kotlin。

## 核心特性

### 1. 任意形状数据结构

Jimmer 的定位是**一个为任意形状数据结构设计的 ORM**，其所有功能都是为了操作任意形状的数据，而非一个个简单的实体对象。

- 实体对象可以是残缺的，任何读写操作都可以只针对对象的部分属性，而非所有属性
- 直接读写任意复杂的数据结构，数据结构的层次和形状由具体业务场景随意指定，无需事先统一固化设计
- 允许某些属性缺失，对象缺少某个属性（值未知）和属性为 null（值已知）是完全不同的两回事

### 2. 动态性与不可变性

Jimmer 实体被声明为 **interface**，而非 class，由预编译器（APT/KSP）负责实现。

**动态性**：
- 在静态语言的优点（高性能、拼写安全、类型安全、空安全）基础上，结合了动态语言的灵活性
- 每个属性都可以缺失，享受静态语言好处的同时，为数据结构赋予绝对的灵活性

**不可变性**：
- Jimmer 对象是不可变对象，保证数据结构一定不会包含循环引用，可以直接被 Jackson 序列化
- 借鉴了 Immer 的思路，基于现有不可变数据结构创建新的数据结构，未修改的局部会复用旧对象

### 3. DTO 语言自动生成

解决了业务模型和数据模型不一致的问题：

- 实体是全局统一的，稳定的；DTO 是针对特定业务的，易变的
- Jimmer 主张集中精力设计高质量实体模型，DTO 可以用极其廉价的方式自动生成
- 在 `src/main/dto` 目录下创建扩展名为 `.dto` 的文件，编译时自动生成 DTO 类型
- 100% 符合静态语言安全性，所有问题在编译时就能发现

### 4. Fetcher - 任意形状查询

这是 Jimmer 三个最基础核心功能之一：

- 像 GraphQL 一样细腻地控制被查询数据的格式
- 支持：查询残缺对象、附带关联对象、递归查询（GraphQL 不支持递归）
- 能处理复杂计算属性，自动批量优化查询性能，没有 N+1 问题
- 支持远程关联，跨越微服务边界的 ORM 关联，自动从不同微服务查询数据组装

### 5. Save Command - 任意形状保存

这是 Jimmer 三个最基础核心功能之二：

- 一个方法调用保存任意形状的数据结构，无论表单多复杂
- 自动对比差异，执行相应的 insert、update、delete
- 支持：保存单对象、保存短关联（只改变关联关系）、保存长关联（级联修改关联对象）、递归保存树形结构

### 6. SQL DSL - 动态查询

这是 Jimmer 三个最基础核心功能之三：

- 天生为任意复杂的动态 SQL 设计，支持动态表连接、隐式子查询
- 智能分页支持，自动生成分页查询
- 支持嵌入原生 SQL 表达式
- 支持超级 QBE，通过 DTO 语言自动生成查询条件

### 7. 批量操作优化

- 充分发挥底层数据库批量修改能力，保存数据结构的每一层都采用 JDBC 批量操作
- 充分利用 upsert 能力，尽可能避免 select 语句
- 对数据库约束错误进行细致翻译，让业务项目无需亲自处理约束异常

### 8. 强大的缓存机制

- 支持多级缓存，每一级都允许用户选择自己喜欢的技术（例子中一级用 Caffeine，二级用 Redis）
- 不只支持 id -> 对象缓存，还支持：id -> 关联 id 缓存、关联属性缓存、id -> 计算结果缓存
- 高度自动化的缓存一致性，只要启用 Trigger 就能自动维护
- 支持多视图缓存，配合权限系统使用

### 9. 其他高级特性

- **Trigger**：数据库变更通知，支持 BinLog 触发器（跨进程感知变化）和 Transaction 触发器
- **Global Filter**：全局过滤器，插件方式添加 SQL 过滤条件，适合行级权限管理
- **Draft Interceptor**：前置拦截器，保存前调整数据
- **Logical Deletion**：逻辑删除，支持实体表和中间关联表
- **Remote Associations**：跨越微服务边界的远程实体关联
- 强类型 DSL 支持，仍可使用特定数据库的特有功能

## 核心优势对比

| 特性 | 传统 ORM (JPA) | Jimmer |
|------|---------------|--------|
| 数据形状 | 必须固定完整 | 任意形状，灵活控制 |
| DTO | 手工编写/半自动化，成本高 | DTO 语言自动生成，极廉价 |
| N+1 问题 | 常见 | 完全解决，即使复杂计算属性 |
| 动态查询 | 单表为主 | 支持任意复杂多表动态查询 |
| 缓存一致性 | 需要手工维护 | 高度自动化 |
| 跨服务关联 | 不支持 | 支持远程关联，自动组装 |

## 预编译技术

Jimmer 采用预编译技术，绝大部分魔法都来自编译时，而非运行时：

- Java: APT (Annotation Processor Tool)
- Kotlin: KSP (Kotlin Symbol Processing)

> 小贴士：如果直接用 IDE 打开未经首次编译的项目，会发现一些类型未定义，这是正常现象，这些类型在预编译后才会自动生成。直接点击 Run 按钮，错误会自动消失。
